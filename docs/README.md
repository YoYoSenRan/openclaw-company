# OpenClaw 可视化经营指挥项目文档总览

## 文档目的
本目录用于沉淀本项目的唯一事实来源（Single Source of Truth）。
后续需求评审、任务拆分、开发实现、测试验收都必须以本目录文档为准，避免方向漂移。

## 阅读顺序（建议）
1. `00-project-charter.md`：项目目标、边界、成功标准。
2. `01-product-design.md`：核心玩法与交互体验。
3. `02-visual-and-animation-spec.md`：像素卡通拟人风格与动画规范。
4. `03-system-architecture.md`：系统架构与数据流。
5. `04-domain-model-and-state-machine.md`：领域模型与状态机。
6. `05-event-protocol.md`：实时事件与命令协议。
7. `06-tech-stack-and-standards.md`：技术栈、目录规范、编码标准。
8. `07-development-workflow.md`：研发流程、测试与发布治理。
9. `08-roadmap-and-milestones.md`：里程碑与排期。
10. `09-risk-and-adr.md`：风险台账与架构决策记录。
11. `10-runtime-config-and-env.md`：运行配置、环境变量与安全边界。
12. `11-openclaw-method-matrix.md`：项目动作与 OpenClaw 方法映射矩阵。
13. `12-test-matrix.md`：测试矩阵、门禁与验收基线。

## 文档治理规则

### Scope 约束
- 任何新需求必须先更新文档，再进入代码实现。
- 功能实现不得超出对应文档章节定义的范围。

### 变更流程
1. 提交变更提案（说明影响文档编号与章节）。
2. 完成文档修订并评审通过。
3. 基于文档创建开发任务。
4. 开发完成后反向校验文档与实现一致性。

### 引用规范
- 任务单必须引用文档章节编号，例如：`Ref: 05-event-protocol.md#3.2`。
- 代码 PR 描述必须包含对应文档引用。
