# 00. 项目总纲（Project Charter）

## Scope
本项目旨在构建一个“OpenClaw Agent 公司化可视系统”：
- 视觉上是俯视角像素卡通办公室。
- 玩法上结合“模拟经营”与“指挥中心调度”。
- 技术上与 OpenClaw Gateway 实时联动，员工即 Agent，行为与状态来自真实事件流。

## Non-goals
- 不做 MMO/多人联网对战。
- 不做复杂 3D 引擎渲染。
- 不做通用低代码平台。
- 不替代 OpenClaw 本体能力，仅做可视化编排与运营层。

## Out-of-scope（当前阶段）
- 商业化计费系统。
- 完整权限中心（RBAC 多租户）。
- 多组织跨公司协同功能。

## 产品愿景
将抽象的 AI Agent 运作状态转化为可观察、可操控、可复盘的“公司运转画面”，
让使用者在“像玩游戏一样调度团队”的体验中完成真实生产任务。

## 目标用户
1. 个人开发者 / 独立创作者：希望可视化管理多个 Agent。
2. 小团队负责人：需要实时掌握任务分发、阻塞和执行效率。
3. 演示与传播场景：需要可观赏的“AI 公司运行大屏”。

## 价值主张
1. 看得见：Agent 状态与协作关系可视化。
2. 管得动：支持实时指令与批量调度。
3. 能复盘：支持时间线回放与故障还原。

## 核心体验原则
1. 可读性优先：状态表达必须一眼可懂。
2. 响应即时：关键指令反馈延迟可感知上限 300ms（前端感知层）。
3. 演出有意义：动画必须映射业务事件，不做纯装饰。
4. 真实可控：可随时从“观赏”切换到“调度操作”。

## 业务目标（第一阶段）
1. 支持 20~50 个 Agent 的实时状态展示。
2. 支持单体与批量任务调度（assign/pause/resume/retry/handoff）。
3. 支持基础经营变量（精力、专注、压力、技能）驱动效率变化。
4. 支持最近 5~15 分钟事件回放。

## 依赖前提
1. OpenClaw Gateway 版本需与 `11-openclaw-method-matrix.md` 的已验证映射一致。
2. Node 运行时基线遵循 `>=22.12`。
3. 未通过契约校验的方法禁止进入实现阶段。

## 成功标准（KPIs）
1. 状态同步成功率 >= 99.5%。
2. 用户操作到画面反馈 P95 < 300ms（局域网/同机房）。
3. 新用户首次完成“派发任务 -> 查看执行 -> 处理阻塞”路径 <= 5 分钟。
4. 连续运行 8 小时无致命崩溃。

### KPI 统计窗口
- 稳定性指标默认按“单实例连续运行窗口”统计。
- 断连恢复可用性单独统计：在网络抖动场景下 5 秒内恢复为达标。

## 关键角色与职责
1. Product Manager：定义任务策略、优先级、验收口径。
2. Operator（玩家/调度者）：实时下达与调整指令。
3. Agent（员工）：执行任务并上报状态事件。
4. System（中间层）：负责事件聚合、状态计算、协议桥接。

## 里程碑定义
1. M1（可视基建）：地图、角色、状态图标、实时连接可运行。
2. M2（指挥中心）：可调度、可中断、可告警。
3. M3（经营系统）：数值模拟、效率波动、事件剧情。
4. M4（复盘系统）：时间轴回放、故障定位、性能面板。

## 验收基线
- 每个里程碑必须具备“演示路径”与“自动化/半自动测试清单”。
- 每个新增特性必须同时满足：产品场景定义、协议定义、状态机定义、UI 表达定义。

## 版本策略
- `v0.x`：快速迭代，允许协议增量调整。
- `v1.0`：冻结核心协议与状态机，进入稳定发布周期。
